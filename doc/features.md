# Модули приложения

Краткое описание каждого модуля и его ключевых бизнес-правил. Полные приёмочные критерии —
[specs/001-timefocus-mvp/spec.md](../specs/001-timefocus-mvp/spec.md).

## Трекер активностей (`features/tracker`)

Сетка иконок активностей → тап запускает трекинг, карточка с тикающим таймером появляется
вверху списка. Пауза/возобновление/стоп. Несколько обычных активностей могут идти
параллельно. Активности-группы разворачиваются in-place (кнопка «Назад» вместо перехода на
отдельный экран).

Матрица переходов при старте новой активности определена в **одном месте** —
`StartActionUseCase`:

- активность с `mode == pomodoro` или `mode == breakFor`, либо флагом `pauseOthers` —
  прерывает текущий Помодоро-интервал (`interrupted`) и ставит остальные активности на паузу;
- системное завершение рабочего интервала Помодоро **не** прерывает сам Помодоро;
- второй запуск Помодоро при уже активном требует подтверждения пользователя.

## Помодоро (`features/pomodoro`)

Циклы работа → перерыв, 3 типа перерывов (short/long по номеру цикла в серии), настраиваемое
число циклов (3–5), действие после серии (`afterAction`, 5 вариантов). Интервал засчитывается
как `completed` только если завершился системно; `interrupted` не сбрасывает номер цикла —
только явный «Стоп» активности.

## Вода и HUD (`features/water`)

Всегда видимая HUD-панель (в shell, поверх всех вкладок): полоска выпитого/цели с меткой
ожидаемого объёма к текущему моменту, кнопка-стакан (тап — стандартная порция, long-press —
список напитков), контекстная иконка.

Два режима напоминаний:
- **interval** — следующее уведомление = `lastDrankAt + N минут`, пересоздаётся после каждого
  питья;
- **scheduled** — все уведомления дня планируются заранее, питьё не сбрасывает расписание;
  автопропуск в окне ±15 минут вокруг приёмов пищи, отбрасывание после окна сна.

Приоритет контекстной иконки: `Туалет (4) > Еда (3) > Спорт (2) > Сон (1) > пусто (0)`.
Туалет показывается только при включённых флагах `showToiletOnWater` / `showToiletOnBreak`.

## Расписание дня (`features/schedule`)

Вертикальный таймлайн на день с дорожками для параллельных интервалов: план (события),
факт (запуски активностей), вода, напоминания. События двух типов:

- **строгие** (`mealStrict` и т. п.) — если до начала не хватает времени на текущий
  Помодоро-интервал, предупреждение приходит немедленно; в точное время события Помодоро
  принудительно прерывается, активность стартует только по тапу на уведомление;
- **гибкие** — ждут завершения текущего Помодоро-интервала, пульсация HUD вместо прерывания.

## Уведомления (`features/notifications`)

Единая система на 9 типов (`NotificationType`), выше остальных модулей не имеет прямых
зависимостей — координация через `RootBlocListener`. Payload содержит всё необходимое для
разбора при холодном старте приложения (`{'actionId', 'breakActionId', 'pomodoroCount', ...}`).
Отложенные уведомления (Помодоро активен, мьют) копятся и доставляются все по очереди после
снятия блокировки. Разрешения зависят от версии Android (`POST_NOTIFICATIONS` с SDK 33,
`SCHEDULE_EXACT_ALARM` с SDK 31, с fallback на неточное планирование).

## История и отчёты (`features/history`)

Режимы день/неделя/месяц/произвольный период с навигацией стрелками и «сегодня». Шапка:
суммарное время (без активности «Сон»), статус Помодоро (завершено/прервано), вода
(выпито/цель). Редактирование интервала — быстрые кнопки сдвига времени, инлайн-валидация
(конец раньше начала — блокирует сохранение), пересечение интервалов одной активности —
предупреждает, но не блокирует. Экран отчётов — 7 пресетов периодов + графики (`fl_chart`).

## Настройки (`features/settings`)

Раздел «Больше»: системные настройки (тема/язык/формат времени — применяются мгновенно),
редактор активностей (создание/правка, FontAwesome-иконка, цвет, режим, тип Помодоро;
системные активности нельзя удалить, только архивировать), настройки Помодоро/воды/
расписания (сохранение = новая версия настроек, не перезапись).

## Онбординг (`features/onboarding`)

Пропускаемый первый запуск: описание функций, опциональный ввод имени, запрос разрешения на
уведомления с корректной обработкой отказа. После `skip` приложение полностью работоспособно
с 12 предустановленными системными активностями; повторный запуск онбординг не показывает.
