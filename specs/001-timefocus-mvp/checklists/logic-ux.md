# Чеклист качества требований: переходы, уведомления, UX

**Purpose**: «Unit-тесты для требований» — проверка полноты, ясности и непротиворечивости
спецификации в зонах: матрица переходов и координация модулей, система уведомлений, UX экранов.
Проверяется КАЧЕСТВО ФОРМУЛИРОВОК в spec.md, а не работа реализации.
**Created**: 2026-07-18
**Feature**: [spec.md](../spec.md)

- [x] CHK002 - Однозначно ли определено, что происходит с обычной (nothing) активностью, когда стартует активность с pauseOthers: останавливается только Помодоро-активность или все запущенные? [Ambiguity, Spec §FR-010]

- [x] CHK004 - Определено ли, что происходит с breakFor-активностью и перерывом, если во время перерыва стартует pauseOthers-активность (туалет во время перерыва)? [Gap, Coverage]
- [x] CHK005 - Согласованы ли требования паузы Помодоро-активности (FR-018a: interrupted) со сценарием возобновления: указан ли номер цикла нового интервала (продолжается или сбрасывается)? [Consistency, Spec §FR-018a]
- [x] CHK006 - Определено ли поведение «Стоп» Помодоро-активности (в отличие от паузы): статус интервала и сброс счётчика цикла? [Gap, Spec §FR-017]
- [x] CHK007 - Специфицировано ли, к какой активности возвращается пользователь после завершения перерыва, если родительская Помодоро-активность была остановлена во время перерыва? [Edge Case, Gap]
- [x] CHK008 - Определено ли поведение авто-действий (autoStartBreak/autoStartWork), когда приложение закрыто в момент срабатывания (действие выполняется при следующем открытии или по тапу)? [Ambiguity, Spec §FR-018]
- [x] CHK009 - Есть ли требования к одновременному наступлению двух прерывающих событий (строгое событие в момент конца рабочего интервала)? [Edge Case, Gap]
- [x] CHK010 - Согласовано ли требование «Туалет прерывает Помодоро» (US3 сценарий 7) с состоянием перерыва — прерывается ли что-то, если тап по виджету произошёл во время перерыва? [Consistency, Spec §FR-028]

- [x] CHK020 - Определено ли, пересоздаются ли запланированные уведомления после перезагрузки устройства (reboot), и чьей ответственностью это является? [Gap, Critical]
- [x] CHK021 - Есть ли требования к идентификации/замене дублирующихся уведомлений одного типа (второй waterReminder до доставки первого)? [Gap, Spec §SC-007]
